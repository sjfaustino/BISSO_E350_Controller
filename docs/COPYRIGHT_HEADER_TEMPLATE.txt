## Script to Add Headers Automatically

### add_copyright.sh (Bash)

```bash
#!/bin/bash
# Script to add copyright headers to source files

YEAR="2025"
AUTHOR="Sergio Faustino"
EMAIL="sjfaustino@gmail.com"
LICENSE="MIT License"  # Change as needed
PROJECT="BISSO E350 Bridge Saw Controller"
VERSION="1.0.0"

add_cpp_header() {
    local file=$1
    local filename=$(basename "$file")
    local brief="$2"

    local header="/**
 * @file $filename
 * @brief $brief
 *
 * @copyright Copyright (c) $YEAR $AUTHOR
 * @license $LICENSE
 *
 * @project $PROJECT
 * @version $VERSION
 * @date $(date +%Y-%m-%d)
 * @author $AUTHOR <$EMAIL>
 */

"
    # Backup original
    cp "$file" "$file.bak"

    # Add header
    echo "$header" | cat - "$file" > temp && mv temp "$file"
    echo "Added header to $file"
}

# Usage example
# add_cpp_header "src/motion.cpp" "Motion control implementation"
```

- No copyleft (derivatives can be proprietary)
- Simple and well-understood

**Use when:** You want maximum adoption and don't care if others close-source derivatives

### Option 2: Apache 2.0 (Better Patent Protection)

**Pros:**
- Permissive like MIT
- Includes explicit patent grant
- Better for commercial use
- Well-understood legally

**Use when:** You want patent protection but still allow commercial use

```
Licensed under the Apache License, Version 2.0 (the "License");
you may not use this file except in compliance with the License.
You may obtain a copy of the License at

    http://www.apache.org/licenses/LICENSE-2.0
```

### Option 3: GPL-3.0 (Copyleft)

**Pros:**
- Ensures derivatives remain open source
- Strong community protection
- Prevents proprietary forks

**Use when:** You want to ensure all derivatives stay open source

```
This program is free software: you can redistribute it and/or modify
it under the terms of the GNU General Public License as published by
the Free Software Foundation, either version 3 of the License, or
(at your option) any later version.
```

- Minimal legal overhead
- Good for building business relationships

## Adding Headers to Existing Files

### Manual Process (Small Projects)

1. Open each .h/.cpp file
2. Add header template at top
3. Fill in brief description
4. Save file

### Semi-Automated (Medium Projects)

```bash
#!/bin/bash
# Add headers to all source files

for file in src/*.cpp include/*.h; do
    if ! grep -q "Copyright (c)" "$file"; then
        echo "Adding header to $file"
        # Use editor to add header
        # OR use sed/awk to insert at line 1
    else
        echo "Skipping $file (already has copyright)"
    fi
done
```

### Fully Automated (Large Projects)

Use tools like:
- `licenseheaders` (Python package)
- `addlicense` (Go tool)
- Custom script (see add_copyright.sh above)

## Important Notes

1. **Add LICENSE file first** - Headers reference the license
2. **Be consistent** - Use same format for all files
3. **Update year** - When modifying files in new year
4. **Don't overwrite** - Check if header already exists
5. **Backup first** - Always backup before bulk operations
6. **Test compile** - Ensure headers don't break builds
